# Styled Components CSS Variables in composed pattern for React Devs

I always love to make use of modularization pattern in my development phases 🔥

> Modular programming is the process of subdividing a computer program into separate sub-programs and then recombine it.

## Why though?

As a React developer, you might be thinking that you don't need variables in CSS. You have an entire JavaScript engine at your disposal!

There are two reasons to switch to CSS variables in your React app:

1. The ergonomics are nice.
2. It unlocks possibility of reusability!

Here's what colors variables look like in variables directory:

```jsx
const colors = {
  '--color-error': '#FF6347',
  '--color-success': '#228B22',
  '--color-warning': '#CCCC00',
}

export default colors
```

Here's what sizes variables look like in variables directory:

```jsx
const sizes = {
  '--small-text': '1em',
  '--medium-text': '2em',
  '--large-text': '4em',
}

export default sizes
```

Here's what CSS variables look like:

```jsx
import { createGlobalStyled } from 'styled-components'
import colors from './variables/colors'
import sizes from './variables/sizes'

// holds global css variables
export const GlobalStyles = createGlobalStyled`
    html{
        --font-weight: 600;
        ${{ ...colors }}
        ${{ ...sizes }}
    }
`
```
